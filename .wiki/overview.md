# AROS Pipeline Factory: Overview

## Mission
The AROS (Antigravity Research OS) Pipeline Factory is a meta-repository designed to forge, audit, and deploy high-fidelity scientific workflows. Its mission is to assist humans in the **Cognitive Decomposition** of their intellectual activities, allowing complex research tasks to be precisely described and outsourced to AI agents.

## Philosophy: Embodiment & Co-Evolution
The foundational philosophy of AROS rests on four pillars:

1. **The Embodiment Gap**: Recognizing the fundamental distinction between discrete digital computation (AI) and continuous, thermodynamic, analog computation (biological life).
2. **Scientific Grounding**: Leveraging frontier research in sensory perception (Remote Touch), stochastic resonance, and interoception to understand biological intelligence.
3. **Cognitive Decomposition**: Treating human-AI interaction as a system where humans define the strategy and modular steps, while AI handles the execution.
4. **Memory Transfer Learning (MTL)**: Harnessing cross-domain meta-knowledge (abstractions, validation routines) to enable agents to self-evolve beyond single-domain silos.
5. **Experience-Driven Skill Curation (SkillOS)**: Decoupling skill curation from task execution, enabling modular, RL-trained self-evolution where a learned curator manages the skill lifecycle (insert/update/delete) optimized by downstream task performance.

For the full manifesto, see the project [README.md](../README.md).

## Core Architecture
AROS operates on a modular "Pipeline" architecture. Each pipeline is a standalone unit of scientific production (e.g., Grant Writing, Image Processing) that shares a common library of Skills, KIs, and Policies via the **Shared Asset Management System (SAMS)**.

## Self-Evolution Roadmap
Based on a rigorous gap analysis against MTL and SkillOS research, AROS is pursuing a 5-phase integration roadmap:
1. **Phase 1**: Insight Memory Layer — Extract domain-agnostic meta-knowledge alongside existing facts/models.
2. **Phase 2**: Full Skill Lifecycle — Enable update/delete operations (not just insert) via GEPA.
3. **Phase 3**: Curator/Executor Decoupling — Formally separate skill curation agents from task execution agents.
4. **Phase 4**: RL-Trained Curation — Replace heuristic evolution with GRPO-trained Skill Curator.
5. **Phase 5**: Agentic Retrieval — Multi-hop, reformulation-capable skill retrieval.
