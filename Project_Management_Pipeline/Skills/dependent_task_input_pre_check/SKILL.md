# Dependent Task Input Pre-Check

This skill provides a standardized pre-flight check for any task that relies on file-based inputs from upstream dependencies. It ensures that required files exist, are readable, and have a valid structure before the core task logic begins, preventing common runtime failures.

## Input Validation

This skill accepts a list of file paths as input for validation.

If the user's request does not involve validating the existence and integrity of file-based artifacts, do not proceed with this workflow. Instead respond:
> "The 'dependent_task_input_pre_check' skill is designed to validate file-based dependencies. Your request does not appear to involve file paths. Please provide one or more file paths to check, or use a more appropriate tool for your task."

## When to Use

- In any workflow where a task consumes files generated by a preceding task.
- Before running computationally expensive processes that depend on valid input files.
- As a gating mechanism in CI/CD pipelines for data processing.
- To provide clear, early feedback to users about missing or corrupt input data.

python -m py_compile scripts/main.py

## Workflow

1.  **Receive a list of file paths** to be validated.
2.  For each file path, perform the following checks sequentially:
    a.  **Existence**: Verify the file exists at the given path (`os.path.exists`).
    b.  **Accessibility**: Verify the file has read permissions (`os.access`).
    c.  **Integrity (Size)**: Verify the file is not empty (`os.path.getsize > 0`).
    d.  **Integrity (Format)**: If the file is a `.json` or `.csv`, attempt to parse it to ensure it's well-formed. Other file types are skipped.
3.  **Aggregate results**: Collect the validation status for each file.
4.  **Return structured output**: Provide a JSON report detailing the success or failure of each check for every file, along with an overall status.

## Core Capabilities

The script can validate three key aspects of a file dependency:
1.  **Existence & Accessibility**: Checks if the file is present and readable.
2.  **Basic Integrity**: Ensures the file is not empty.
3.  **Format Integrity**: For supported types (JSON, CSV), it performs a shallow parsing to validate the structure.

## Usage

### Validate a Single File
```bash
python scripts/main.py /path/to/my_data.csv --json-output
```

### Validate Multiple Files
```bash
python scripts/main.py /path/to/data.json /path/to/another/input.txt --json-output
```

### Example JSON Output (Success)
```json
{
  "overall_passed": true,
  "results": [
    {
      "filepath": "/path/to/data.json",
      "exists": true,
      "accessible": true,
      "integrity": {
        "non_empty": true,
        "format_valid": true,
        "error": null
      }
    }
  ]
}
```

### Example JSON Output (Failure)
```json
{
  "overall_passed": false,
  "results": [
    {
      "filepath": "/path/to/nonexistent_file.csv",
      "exists": false,
      "accessible": false,
      "integrity": {
        "non_empty": false,
        "format_valid": null,
        "error": "File does not exist."
      }
    },
    {
      "filepath": "/path/to/corrupt.json",
      "exists": true,
      "accessible": true,
      "integrity": {
        "non_empty": true,
        "format_valid": false,
        "error": "Invalid JSON format: Expecting ',' delimiter: line 2 column 5 (char 15)"
      }
    }
  ]
}
```
