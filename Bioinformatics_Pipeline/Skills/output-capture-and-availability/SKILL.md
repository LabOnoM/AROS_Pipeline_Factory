# Output Capture and Availability Policy

This document outlines the mandatory policy for capturing all outputs generated by AROS agents. The primary objective is to ensure that every piece of generated data—from code snippets to tool outputs and final responses—is logged, stored, and made available for subsequent analysis, debugging, and workflow continuity.

## GEPA Rules

**GEPA-Rule-001: Mandatory Output Interception**

All outputs generated by an agent, including but not limited to final responses, intermediate thoughts, generated code, and tool calls, **MUST** be intercepted by the execution environment before being displayed to the user or passed to another tool. Silent or unlogged outputs are a direct violation of this policy.

**GEPA-Rule-002: Structured Output Logging**

Each intercepted output **MUST** be recorded as a structured log entry. The log entry must contain the following minimum fields:
-   `timestamp`: The ISO 8601 timestamp of the event.
-   `agent_id`: The unique identifier of the agent generating the output.
-   `session_id`: The identifier for the current user interaction or task chain.
-   `output_type`: The nature of the output (e.g., `thought`, `tool_code`, `shell_command`, `final_response`).
-   `content`: The complete, unmodified output data.
-   `metadata`: A JSON object for additional context, such as the tool name, source file, or line number.

**GEPA-Rule-003: Persistent and Addressable Storage**

All structured output logs **MUST** be written to the designated persistent storage location (`~/.gemini/antigravity/logs/output_capture.log`). Each log entry must be implicitly addressable (e.g., by line number or a unique transaction ID) to facilitate later retrieval.

**GEPA-Rule-004: Ensure Downstream Availability**

The system **MUST** provide a mechanism for subsequent agents or tools in a workflow to query and retrieve the outputs of prior steps using the `session_id` or another correlation identifier. This prevents data silos between steps and ensures that context is passed reliably.

## Rationale

This policy is critical for the stability, observability, and evolution of the AROS ecosystem. By capturing all outputs, we enable:
-   **Traceability and Debugging**: A complete record of agent actions allows for precise debugging of failures.
-   **Performance Analysis**: Systematically logging outputs provides the raw data needed for performance analysis and optimization.
-   **Knowledge Distillation**: The captured logs serve as a primary source for the GEPA mutation engine to discover new patterns and propose system improvements.
-   **Reproducibility**: Ensures that complex agent behaviors can be reviewed and reproduced.
