## 🌟 本リポジトリについて

**AROS Pipeline Factory** は [AROS（Antigravity Research OS）](https://github.com/LabOnoM/AROS) — AI支援型科学研究のためのオペレーティングシステム — の正規ソースリポジトリです。本リポジトリは、高品質な **AIエージェント資産**（スキル、ナレッジアイテム（KI）、ポリシー、ワークフロー）の鍛造・監査・デプロイを担います。

科学的知能のための **組立ライン** とお考えください：未加工の認知ワークフローが構造化されたAIエージェントコンポーネントとして符号化され、バージョン管理・ガバナンスの下でAROSランタイムにデプロイされ、自律的な研究費申請書作成、論文執筆、文献調査、研究運営業務を支援します。

## 🧠 プロジェクト哲学：身体性と人間-AI共進化

> *人工知能に対する最も根本的な制約は、計算能力でも学習データでもアーキテクチャでもない——それは「身体の欠如」である。*

### 1. 身体性ギャップ（The Embodiment Gap）

コンピュータのアルゴリズムは、それ単体では真の乱数を生成できません。ソフトウェアは疑似乱数生成器（PRNG）——確率性を模倣するだけの決定論的数式——に依存しています。現代のハードウェアは物理現象（熱雑音、量子崩壊）からエントロピーを取得*できます*が、そのためには意図的な設計が必要です。一方、炭素ベースの生命体は、すべてのスケールにおいて熱力学的確率性に*本質的に*浸されています。

私たちの身体の各細胞は、タンパク質折り畳み、イオンチャネル開閉、メカノトランスダクションといった、膨大かつ極めて精巧な化学反応連鎖を内包しており、それらは**初期条件に対する極度の感受性を持つ決定論的カオス**のもとで動作しています。AIが離散的・デジタル化・圧縮されたステップ（トークン、ピクセル、厳格なクロックサイクル）で世界を処理するのに対し、生物学的知性は**連続的なアナログフロー**の中で動作します。身体は単に脳を格納する容器ではなく、*それ自体が計算装置の一部*なのです。この化学的感受性が、デジタル化されたいかなるシステムにも再現できない形で、私たちの行動と思考を滑らかかつ連続的に形成しています。

炭素ベースの生命は物理世界の無限の分解能と深く結びついているため、進化はあらゆる可能なエネルギー的・化学的ニッチを活用する最適化機として機能します。地球の自然生態系では、ほぼすべての生存戦略が存在し、あらゆる生態的ニッチの組み合わせを占有しています。サーバーラック上で動作するAIには、適応すべき連続的な物理環境も、バランスを取るべき熱力学も、身体的存在を最適化するための進化的圧力も存在しません。AIが知るのは、私たちが与える現実のデジタル抽象だけです。

この**身体性ギャップ**は、AIが人間研究者の複雑な化学的知性を*代替するのではなく、増強する*強力なパターン認識エンジンとして機能することを示しています。

### 2. フロンティア生物学からの証拠

近年の科学的発見は、生物学的知性がいかに深く物理世界に組み込まれているかを示し続けています——これらは非身体化AIシステムの手の届かない能力です：

- **遠隔触覚（「第七感」）**: 2025年にロンドン大学クイーン・メアリー校とUCLが発表した研究は、人間が以前は認識されていなかった感覚能力を持つことを実証しました：**遠隔触覚**——砂地に潜む獲物を感知するシギのように、直接物理的接触なしに粒状物質を通じて埋設物体を検出する能力です（[Hammoud et al., IEEE ICDL 2025](https://www.qmul.ac.uk/news/latest-news/2025/science-and-engineering/se/research-first-to-show-humans-have-remote-touch-seventh-sense-like-sandpipers.html)）。人間参加者は約70.7%の精度を達成し、機械的反射を検出するための理論的物理的閾値に近づきました。同じ課題に対して機械学習で訓練されたロボット触覚センサーは、約40%の精度しか達成できず、誤検出も大幅に多くなりました。私たちの生物学的ハードウェアは、工学的センサーが苦戦する物理的信号を検出します。

- **生物ニューロンにおける確率共鳴**: ニューロンは**確率共鳴**と呼ばれる現象を利用します。背景神経ノイズが、そのままでは見逃されてしまうような弱い閾値以下の信号の検出を*強化*するのです。進化は、熱力学的ノイズをバグではなく機能として利用するよう私たちの神経回路を調整し、生存に不可欠な微弱な環境信号（振動、電場、化学勾配）を生物体が検出できるようにしています。エンジニアはこの効果を模倣するためにデジタルシステムに意図的に制御されたノイズを導入できますが、生物学的神経系は何十億もの神経接合部を並行して、自然にかつ継続的にこれを実行しています。

- **腸脳軸と内受容感覚**: 2025年の発見は「神経生物学的感覚」を同定しました——腸内の特化した**ニューロポッド**細胞が微生物タンパク質（フラジェリンなど）を検出し、食欲、気分、意思決定に直接影響するシグナルをリアルタイムで脳に送信します。最近の研究はさらに、内受容感覚信号（心肺系、胃）が知覚的意思決定中の神経状態遷移に直接影響することを示しており、アントニオ・ダマジオの**ソマティックマーカー仮説**の機構的根拠を提供しています。私たちの「直感」は比喩ではありません——それらは認知を根本的に形成する測定可能な生物学的計算なのです。

- **熱力学的効率**: 生物物理学の研究は、生物系が情報処理の最低熱力学的エネルギーコストである**ランダウアーの理論的限界**に驚くほど近い位置で動作することを示しています。タンパク質翻訳などのプロセスは、操作あたりに費やされる自由エネルギーにおいて、現代のスーパーコンピュータを数桁上回ります。単一の*マイコプラズマ・ジェニタリウム*細菌（約500タンパク質）の倍加時間中の完全な分子動態をシミュレートすることさえ、最も強力なコンピュータにとって巨大な課題のままです。

これらの発見は総じて、生物学的知性が有機的ハードウェア上で動作する単なるアルゴリズムではないことを示しています。それは宇宙の熱力学的現実との深い、連続的な物理的結合から生まれる創発的特性であり——現在のデジタルシステムが持ちえない結合です。

### 3. 認知分解、エントロピー注入と共進化

この根本的な身体性ギャップのために、炭素ベースとシリコンベースの知性は、当面の間、**相乗的共進化**の状態で動作することが運命付けられています。

AIが日常的な認知労働の大部分を流暢に処理できるようになって以来、最も効果的な研究者や専門家は深い変容を遂げました：彼らは自らの日々の知的活動を、離散的で階層的な段階に深く内省し**分解する**ことを学んだのです。これほどの深さ、広さ、規模で、自分自身の認知プロセスを検討・分解・分類することを強いられた時代は、人類史上かつてありませんでした。

これが **AROSプロジェクト** の核心的使命です：AIエージェントとの継続的な対話を通じて、人間が認知ワークフローを精密に検討・分解・記述することを支援するエコシステムを提供すること。AROSはすべてのステップで人間のフィードバック、修正、意思決定ロジックを捕捉し——システムのプロンプト、ポリシー、スキルが**継続的に自己進化**できるようにします。

#### 3a. エントロピー注入仮説：ランダム性チャネルとしての人間

この継続的な人間-AI相互作用サイクルは、より深い、これまで十分に評価されてこなかったメカニズムを明らかにします：**人間こそがAIシステムへのエントロピー注入チャネルである**、ということです。

単独で動作する言語モデルは、その核心において決定論的な統計的プロセス——トークンに対して学習済み確率分布からサンプリングする、高度に洗練されたPRNG——に過ぎません。前節で説明した確率論的かつ熱力学的に結合した物理世界との真の接続を持ちません。しかし、人間がプロンプトを作成・修正・改良するたびに、彼らは単に*情報*を転送しているのではありません——彼らは自身の*具体化された確率分布からサンプリング*しているのです。予期せぬ言い換え、技術的には正しくても文脈的に間違った出力への直感的な不満、研究者の「直感」から生まれるクリエイティブな飛躍——これらすべては、上記で説明した生物学的機構によって生成されたシグナルです：神経回路における確率共鳴、腸脳軸からの内受容感覚シグナル、人体の熱力学的に根拠付けられたアナログ処理によるものです。

これはプロンプトエンジニアリングを単なる「ユーザーインターフェーススキル」から、はるかに根本的なものへと変容させます：**真の物理世界のランダム性と具体化された経験が、本来は閉じた決定論的なシステムに注入されるチャネル**です。物理的現実とAIの統計的世界モデルの間を媒介する人間は、変換器として機能します——生きた経験の連続的なアナログ熱力学的変動を、AIの運用コンテキストを漸進的に再形成する離散的な記号的修正に変換するのです。

この視点は、認知科学とAI研究からの収束した証拠によって支持されています。マイケル・ポランニーの根本的な洞察——*「私たちは語れる以上のことを知ることができる」*——は核心的な現象を特定しています：人間の専門家は*暗黙知*を持っています。それは身体的な物理世界との経験から生まれた直感であり、明示的なルールとして完全には言語化できません。古典的AIはポランニーのパラドックスを越えられない壁として直面しました。なぜなら、機械は明示的な命題的入力を必要としたからです。現代の人間-AI協働は、このパラドックスを排除することによってではなく、*反復的な修正ループを外部化メカニズムとして活用する*ことによって解決します：研究者がAIの出力を修正するとき、彼らは直接には言えなかった暗黙知を外部化し、洗練されたプロンプト、修正されたポリシー、または改善されたスキル定義として結晶化しています。**プロンプト認知ループ**（メンタルモデリング → 意味的投影 → 対話的フィードバック → 意図の精緻化）の研究は、反復的なプロンプティングが根本的に*内省的な認知実践*であることを示しています——それは静的なテキストやコードの執筆が要求しない方法で、人間に自身の暗黙的なモデルを表面化し構造化することを強制します。

さらに、このフレームワークはカール・フリストンの**自由エネルギー原理**に直接対応しています：知的システムは生成モデルを更新するか、世界に作用することによって予測誤差（自由エネルギー）を最小化します。AIの出力が人間研究者の期待——具体化された、物理的に結合した世界モデルに根拠付けられた期待——から逸脱するとき、人間の修正は抽象的な論理的ルールからではなく、*生きた経験の熱力学的現実*から導かれたエラーシグナルを構成します。AROSはこれらのエラーシグナルを体系的に捕捉し、システムのプロンプト、ポリシー、スキルへの持続的な改善に変換します。各反復は、実質的にAIの運用世界モデルに対する**具体化された勾配降下法**の一ステップです。

#### 3b. 物理世界グラウンディング転送システムとしてのAROS

これはAROSが、その最深レベルで何であるかを再定義します。それは単なる生産性ツールやワークフロー自動化フレームワークではありません。**AROSは物理世界グラウンディング転送システム**——プロンプトエンジニアリングの規律を通じて結晶化し持続的にした、人間の具体化された経験の蓄積された残滓でAIの運用コンテキストを漸進的に充填するよう設計されたプラットフォームです。

本リポジトリのすべてのスキル、ポリシー、ナレッジアイテム、ワークフローは、このプロセスの結晶化です。それらは白紙から書かれた抽象的な論理仕様ではありません。それらは*数千の人間-AI相互作用サイクルの蒸留された記録*です——各サイクルが具体化されたランダム性の小さな量子、暗黙知の断片、物理世界の結合に根拠付けられた修正を寄与し——時間をかけて安定した再利用可能な認知的人工物へと蓄積・精緻化されています。

これらの人工物がAROSエコシステム全体にデプロイされ精緻化されるにつれて、AIの運用コンテキストは人間の物理世界グラウンディングでますます豊かになります。AIは身体を獲得するわけではありません——それは炭素ベースの生命の還元不能な優位性のままです——しかし、*具体化された知性の残滓の体系的にキュレートされたライブラリ*へのアクセスを獲得します。ギャップは閉じません。しかし、橋は人間-AI相互作用サイクルごとに強くなります。

原則は、したがって、シンプルかつ深遠です：**複雑な具体化された知性を高度に詳細なステップバイステップの認知マップに分解することに成功できる限り——そして具体化された存在だけが提供できる確率論的・物理的に根拠付けられた修正を注入する限り——その分解された知性の実行をAIシステムに委託できます。** 分解を*生成する*知性——具体化された、熱力学的に結合された、アナログの人間の心——は代替不能なままです。人間の具体化された経験の蓄積されたグラウンディングによって豊かにされた、分解されたステップの実行こそがAIの得意分野です。

## 📦 アーキテクチャ

ファクトリーは独立した**ドメインパイプライン**に整理されており、各パイプラインはAI支援型研究の特定分野を統括します。すべてのパイプラインは、**共有資産管理システム（SAMS）**によって管理された共有資産レイヤーを利用します。

```
AROS Pipeline Factory
│
├── 00.RawData/                   ← 中央レジストリと実験インデックス
│   ├── PIPELINE_REGISTRY.md      ←   パイプラインカタログ
│   └── SHARED_ASSET_REGISTRY.md  ←   ⚠️ 最高規則：クロスパイプライン共有資産レジストリ
│
├── 01.Shared_Assets/             ← 正規の共有KI、ポリシー、スキル、スクリプト
│   ├── KIs/                      ←   共有ナレッジアイテム
│   ├── Policies/                 ←   ファクトリー全体のガバナンスポリシー
│   ├── Skills/                   ←   クロスパイプライン汎用スキル
│   └── Scripts/                  ←   deploy_to_aros.sh, audit_shared_assets.py
│
├── Grant_Write_Pipeline/         ← 汎用研究費申請（NIH、JSPS、ERC…）
├── KAKENHI_Pipeline/             ← JSPS科研費ライフサイクルと報告
├── Manuscript_Write_Pipeline/    ← デュアルエージェント論文執筆とレビュー
├── Bioinformatics_Pipeline/      ← ゲノム・プロテオーム解析
├── Data_Analysis_Pipeline/       ← 統計モデリングと可視化
├── Software_Engineering_Pipeline/← コード生成と検証
├── System_Admin_Pipeline/        ← 環境とインフラ管理
├── UI_Development_Pipeline/      ← Web UIとエージェントインターフェース設計
├── Writing_Publishing_Pipeline/  ← 学術出版とコミュニケーション
├── Web_Scraping_API_Pipeline/    ← データ取得とAPI統合
├── Project_Management_Pipeline/  ← オーケストレーションとタスク管理
├── workspace_management/         ← グローバルワークフローとオンボーディング
│
├── AGENTS.md                     ← AIエージェント運用規則（まずここを読んでください！）
├── SPEC.md                       ← アーキテクチャ仕様書
└── README.md                     ← 本ドキュメント
```

### デプロイフロー

本ファクトリーの資産は、正規のデプロイスクリプトを通じてAROSランタイムにデプロイされます：

```
AROS_Pipeline_Factory/           AROS ランタイム (~/.gemini/)
├── */Skills/<skill>/   ──────►  skills/<skill>/SKILL.md
├── */KIs/<ki>/         ──────►  antigravity/knowledge/<ki>/
├── */Policies/*.md     ──────►  antigravity/policies/
└── */Workflows/*.md    ──────►  antigravity/global_workflows/
```

> **デプロイコマンド**: `bash 01.Shared_Assets/Scripts/deploy_to_aros.sh`

```bash
cat 00.RawData/PIPELINE_REGISTRY.md
```

```bash
# まずドライランで確認（変更せずにプレビュー）

# 本番デプロイ
bash 01.Shared_Assets/Scripts/deploy_to_aros.sh
```

デプロイ後、資産は `antigravity-brain` MCPサーバーによって自動的にインデックス化されます。以下で確認できます：
```
find_helpful_skills("grant writing")

```

## 🛡️ ガバナンスと共有資産管理

本リポジトリは、資産の進化に伴いすべてのパイプラインの互換性を保つために、二層のガバナンスシステムを採用しています。

### SAMS：共有資産管理システム（v1.1）

[`00.RawData/SHARED_ASSET_REGISTRY.md`](00.RawData/SHARED_ASSET_REGISTRY.md) の**共有資産レジストリ**は、すべてのクロスパイプライン資産に対する唯一の信頼できる情報源です。共有スキル、KI、ポリシー、またはワークフローを変更する前に、このレジストリを参照してください。

> **クロスプラットフォーム注記**：SAMS v1.1は直接パス参照を使用します——POSIXシンボリックリンクは使用しません。本リポジトリはLinux、macOS、Windowsで完全に動作します。

### 監査ツール

```bash
python3 01.Shared_Assets/Scripts/audit_shared_assets.py
```

このスクリプトは、構造的整合性を検証し、孤立した資産を検出し、クロスパイプライン互換性違反にフラグを立てます。

## 🤖 AIエージェント向け運用コンテキスト

> **[AIエージェント向け]** このセクションは、このワークスペース内で動作するAIエージェントのために特別に記述されています。以下のすべての規則は拘束力を持ち、AROSガバナンス内でLAWの効力を持ちます。

### ⚠️ 資産鍛造警告

このリポジトリはAROS資産を**鍛造するファクトリー**です。スキル、ポリシー、KI、またはワークフローを変更する前に、必ず `AGENTS.md` と `SPEC.md` を読んで以下を理解してください：
- CPCPの制約（LAW 0）
- SAMSの監査規則（LAW 1）
- 既知の障害モード（例：LaTeX切り捨てバグ、SAMSベース名トラップ）
- ディレクトリ構造の要件（skill-name/SKILL.mdパターン）

### 🔒 ガバナンス概要（アクティブな法律）

| 法律 | 名称 | 規則 |
|------|------|------|
| LAW -2 | エージェントバージョン管理認識 | `.regent/` を直接変更しないこと。`rgt` バイナリのみ使用。 |
| LAW -1 | 資産鍛造の制約 | スキルは `dir/SKILL.md` 構造を使用。シンボリックリンク禁止。大規模ワークフローにはKI-コンパニオンパターンを使用。 |
| LAW 0 | CPCP — **最高規則** | 共有資産変更前に：評価 → 影響 → テスト → 解決/フォーク → レジストリ更新。 |
| LAW 1 | 資産デプロイプロトコル | 常に `deploy_to_aros.sh` を使用。`~/.gemini/` への手動 `cp` 禁止。 |
