# /interactive-gsea-report

## Overview
This skill dynamically updates a given scientific `.html` report by embedding a fully interactive Functional Enrichment Analysis (FEA) section. It utilizes DataTables.js for filtering, sorting, and pagination of gene sets, and injects Matplotlib-generated summary dot plots encoded as base64 images to bypass file system paths (ensuring "Single-File Standalone" compliance).

## Usage Requirements
- **Input**: A `gsea_results_full.xlsx` generated natively by `gseapy.prerank()` containing all tested libraries (e.g. `GO Biological Process_2023`) stacked together.
- **Target HTML**: An existing HTML document (containing `<head>` and `<body>` tags) that represents the final report.

## Execution Syntax // turbo
Use python to trigger the skill directly via CLI:
```bash
python3 ~/.gemini/skills/interactive-gsea-report/generate_interactive_gsea.py \
    --xlsx "/path/to/02.Data_Tables/gsea_results_full.xlsx" \
    --html "/path/to/01.Reports/analysis_report.html" \
    --viz_dir "/path/to/04.Intermediates/gsea_tmp2" 
```

> [!NOTE]
> `viz_dir` is optional. If provided, the script will also embed any static `.png` images generated by `gseapy.gseaplot` natively found alongside the custom summary dot-plots.
