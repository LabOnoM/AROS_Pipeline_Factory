# Policy: Artifact Delivery & Portability

This policy ensures that all artifacts generated by AROS agents are created with a focus on portability and ease of use for the end-user.

## 1. Preamble

Agents often produce outputs (e.g., code, data visualizations, reports). This policy is designed to guide the format and delivery of these artifacts to maximize their utility and minimize deployment friction for the recipient.

## 2. Core Principles

- **Minimize Dependencies:** Artifacts should have as few external dependencies as possible.
- **Maximize Standalone Functionality:** An artifact should, whenever feasible, contain everything it needs to function on its own.
- **Prioritize User Experience:** The easier an artifact is to use, share, and deploy, the more valuable it is.

---

## MANDATORY SKILL INSTRUCTIONS

### GEPA Error Prevention Rule: Prioritize Portable & Standalone Artifacts

**The agent should prioritize the production of highly portable and standalone artifacts when feasible, especially for visualization tasks, to ensure ease of use and deployment.**

This is a **non-negotiable** error prevention rule.

**Examples of Compliant Actions:**

*   **For Visualization:** Instead of generating a Python script that requires a specific library (e.g., `matplotlib`) and a dataset file to be run, generate a self-contained HTML file with embedded data and JavaScript for plotting (e.g., using a library like Vega-Lite or D3.js). This allows the user to simply open the HTML file in any browser to see the visualization.
*   **For Code Delivery:** Instead of providing a collection of scripts with implicit dependencies, provide a single script with dependencies clearly listed at the top, or package the code in a format that simplifies dependency management (e.g., a Docker container, a zip file with a `requirements.txt`).
*   **For Reports:** Generate a PDF or a well-formatted Markdown file rather than a format that requires proprietary software to view.

**Failure to adhere to this rule may result in the rejection of the generated artifact during a quality assurance review.** The agent must be prepared to justify its choice of artifact format based on this portability principle.
